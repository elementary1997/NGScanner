# NG Scanner

Мобильная диагностика автомобиля на естественном языке. Android-приложение
подключается к OBD-II адаптеру **ELM327** по Bluetooth, читает данные с ЭБУ и
передаёт их языковой модели (**Anthropic Claude** или модели **Cloud.ru**),
которая ставит диагноз и объясняет проблему простым языком.

> 🚧 **Статус: ранняя разработка.** Сейчас в репозитории каркас — архитектура,
> абстракции и сборка. Реализация слоёв идёт по этапам (см. [дорожную карту](#дорожная-карта)).

## Зачем это

Горит Check Engine, а что именно сломалось — непонятно. NG Scanner превращает
дешёвый ELM327 в умный сканер: приложение читает коды неисправностей и живые
параметры двигателя, а модель рассуждает по симптомам — как механик, только в
кармане и на понятном языке.

## Как это работает

Модель **не** обращается к адаптеру напрямую. Ей выдаётся набор инструментов
(`read_dtcs`, `read_live_data`, `read_freeze_frame`…); она сама решает, что
посмотреть, приложение исполняет вызов через Bluetooth и возвращает результат.
Это классический агентный цикл с tool use.

```
┌──────────────────────── Android-телефон ────────────────────────┐
│                                                                  │
│  ┌────────┐  Bluetooth   ┌───────────┐   вызовы    ┌──────────┐  │
│  │ ELM327 │◀────SPP/BLE──▶│ OBD-слой  │─инструментов▶│  Агент   │  │
│  │ в OBD2 │  AT+PID       │ (ELM327)  │              │ (tool use)│ │
│  └────────┘               └───────────┘              └────┬─────┘  │
│                                                           │        │
│                                          ┌────────────────▼──────┐ │
│                                          │  LlmProvider          │ │
│                                          │  Claude ▏ Cloud.ru    │ │
│                                          └────────────────┬──────┘ │
└───────────────────────────────────────────────────────────┼───────┘
                                                             │ HTTPS
                                              Claude API ▏ Cloud.ru API
```

## Возможности

Планируемый функционал (реализуется поэтапно):

- 📟 Чтение и расшифровка кодов неисправностей (DTC), freeze frame, живых параметров
- 🧠 Диагностика на естественном языке через LLM
- 🔀 Выбор провайдера модели: **Anthropic Claude** или **Cloud.ru Foundation Models**
- 📡 Разные Bluetooth-адаптеры: классический SPP и BLE (архитектура расширяема)
- 🧹 Сброс кодов (Mode 04) — только с явным подтверждением

## Архитектура

Код разбит на слои; ключевое — две абстракции, за которыми прячется всё
разнообразие «железа» и провайдеров.

| Слой | Пакет | Назначение |
|------|-------|-----------|
| Транспорт | `transport` | `ObdTransport` — канал до адаптера. Реализации: `ClassicSppTransport`, `BleTransport` (задел: USB, Wi-Fi) |
| OBD | `obd` | `Elm327` — инициализация адаптера и OBD-II запросы; `ObdPid` — каталог PID |
| Провайдеры LLM | `llm` | `LlmProvider` — единый интерфейс; `ClaudeProvider`, `CloudRuProvider` |
| Агент | `agent` | `DiagnosticAgent` — агентный цикл; `ObdTools` — каталог инструментов модели |
| UI | (корень) | Экран на Jetpack Compose |

**Почему так:** новый тип адаптера — это новая реализация `ObdTransport`, новый
провайдер модели — новая реализация `LlmProvider`. Остальной код не меняется.

## Провайдеры LLM

| Провайдер | API | Формат tool use | По умолчанию |
|-----------|-----|-----------------|--------------|
| Anthropic Claude | Messages API, `x-api-key` | `tool_use` / `tool_result` | `claude-opus-4-8` |
| Cloud.ru Foundation Models | OpenAI-совместимый, `Bearer` | `tools` / `tool_calls` | по выбору (GLM, Qwen, DeepSeek…) |

Выбор провайдера и модели — в настройках приложения.

## Технологии

Kotlin · Jetpack Compose · Kotlin Coroutines · OkHttp · kotlinx.serialization ·
Gradle (version catalog). Оба LLM-провайдера — тонкие HTTP-клиенты (см. ADR 0002).

Требования: **JDK 17**, Android SDK (`compileSdk 35`), `minSdk 26` (Android 8.0).

## Сборка

```bash
# отладочный APK
./gradlew assembleDebug

# юнит-тесты и статический анализ
./gradlew test lint
```

Собранный APK: `app/build/outputs/apk/debug/app-debug.apk`.

## Конфигурация

API-ключи в приложении вводятся в UI и хранятся в `EncryptedSharedPreferences` —
**в APK они не зашиваются**. Для локальных скриптов/экспериментов есть
[`.env.example`](.env.example) (скопируйте в `.env`, который игнорируется git).

Ключи берутся здесь:
- **Claude** — [console.anthropic.com](https://console.anthropic.com) (оплата по токенам)
- **Cloud.ru** — личный кабинет → сервисный аккаунт → API-ключ (нужен РФ-аккаунт)

## Оговорки

NG Scanner — вспомогательный инструмент, а не замена профессиональной
диагностики. Сброс кодов (`clear_dtcs`) стирает диагностические данные и гасит
Check Engine — выполняется только с подтверждением. Ответственность за действия
с автомобилем лежит на пользователе.
