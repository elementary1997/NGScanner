# ADR 0001. Ключевые архитектурные решения

- Статус: принято
- Дата: 2026-07-09

## Контекст

NG Scanner — Android-приложение для диагностики авто через OBD-II адаптер ELM327
и LLM. На старте выбирали способ доставки «мозга» на телефон, формат работы с
адаптером и с провайдерами моделей.

## Решения

### 1. Своё приложение + Claude API (а не подписка + MCP)

Рассматривались три пути: (а) своё приложение через Claude API, (б) MCP-сервер +
Claude Code в терминале на подписке, (в) Claude Desktop на ПК. Ограничения:
мобильное приложение Claude не поддерживает custom MCP-коннекторы, подписка не
даёт программного доступа сторонним приложениям, «без Raspberry Pi» требует всё
держать на телефоне.

**Выбрано:** нативное приложение с прямым доступом к Claude API. Даёт свой
мобильный UX и нативный Bluetooth без обходов. Цена — оплата по токенам (для
личного использования — центы за сессию).

### 2. Абстракция транспорта `ObdTransport`

Дешёвые ELM327 бывают на классическом Bluetooth (SPP/RFCOMM) и на BLE (GATT).

**Выбрано:** интерфейс `ObdTransport` с реализациями под каждый транспорт
(`ClassicSppTransport`, `BleTransport`; задел под USB и Wi-Fi). OBD-слой работает
поверх и не знает деталей — один код обслуживает разные адаптеры.

### 3. Абстракция провайдера `LlmProvider`

Нужны Anthropic Claude (Messages API, `tool_use`/`tool_result`) и Cloud.ru
Foundation Models (OpenAI-совместимый API, `tools`/`tool_calls`).

**Выбрано:** интерфейс `LlmProvider` с нейтральной моделью данных и каталогом
инструментов (`ToolSpec`). Каждый провайдер конвертирует инструменты в свой
формат. Агентный цикл не зависит от провайдера; выбор — в настройках.

### 4. Нативный Android (Kotlin + Compose)

Классический Bluetooth SPP надёжнее всего работает через нативный
`BluetoothSocket` (без рута), а Anthropic поставляет официальный Java SDK,
пригодный из Kotlin. Кроссплатформенные фреймворки для classic SPP менее зрелы.

## Последствия

- Диагностический цикл переносим между провайдерами почти без изменений.
- Поддержка нового адаптера/провайдера — это одна новая реализация интерфейса.
- API-ключи требуют безопасного хранения на устройстве (`EncryptedSharedPreferences`).
