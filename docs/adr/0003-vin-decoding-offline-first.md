# ADR 0003. Разбор VIN: офлайн-первый, детали ВАЗ из vininfo, vPIC-резерв

- Статус: принято
- Дата: 2026-07-09

## Контекст

По VIN нужно получать марку/модель/год/двигатель — для карточки машины в
«Гараже» и как контекст диагностическому агенту. VIN пользователь может ввести
руками или считать с ЭБУ (Mode 09 PID 02), поля формы редактируемы, так что
данные — это подсказка, а не источник истины.

Рассмотренные варианты источника:

- **Платные российские агрегаторы** (Автокод, SpectrumData, avtoapi /
  АвтоПроверка). Дают максимум (комплектация, история, ограничения), но требуют
  API-ключа, денег (отчёты «от 17 ₽»), B2B-договора и сети; у части — асинхронная
  task-модель с поллингом. Избыточно для задачи «подсказать поля формы».
- **NHTSA vPIC** — бесплатный публичный REST без ключа, но российские VIN
  (WMI `XTA`, `XTT`, `XTH`…) почти не знает.
- **Библиотека kVinInfo** (Kotlin Multiplatform, MIT) — базовый разбор
  WMI/год/checksum, но общая WMI-база без специфики ВАЗ.
- **Библиотека vininfo** (Python, BSD-3-Clause) — детально знает АвтоВАЗ: по VDS
  извлекает модель, двигатель, кузов, завод (напр. `XTAGFK330JY144213` → Vesta,
  двиг. 21179, Ижевск).

Специфика российского рынка (ВАЗ) — продуктовый моат проекта, и она должна жить
в коде, а не за платным сторонним сервисом.

## Решение

**Офлайн-первый разбор**, реализация — в существующем
[`garage/VinDecoder.kt`](../../app/src/main/java/ru/ngscanner/garage/VinDecoder.kt),
без нового слоя и без новых зависимостей:

- **Марка** — по офлайн-таблице WMI (упор на РФ и Китай).
- **Детали ВАЗ** — для WMI `XTA` офлайн-разбор VDS (модель, двигатель). Таблицы
  кодов **портированы из vininfo** (`details/avtovaz.py`, BSD-3) — реальные
  заводские соответствия, не эвристика. Поле заполняется только при известном
  коде; неизвестный код оставляет поле пустым — ничего не выдумываем.
- **Год** — по 10-й позиции, с поправкой на то, что ВАЗ и китайские марки не
  соблюдают US-конвенцию 7-й позиции (иначе год занижался на 30 лет).
- **Иномарки** — резервный онлайн-запрос к бесплатному NHTSA vPIC (без ключа).

Что сознательно **не** делаем:

- Не вводим платный API (avtoapi/SpectrumData) — при будущей потребности в
  комплектации/истории это отдельный провайдер, не ломающий текущий разбор.
- Не тянем зависимость kVinInfo — ручной разбор WMI/года уже есть и достаточен.
- Не валидируем VIN жёстко по контрольной цифре (SAE, позиция 9): российские VIN
  её штатно не проходят (подтверждено эталонным тестом vininfo), строгая проверка
  отсекла бы валидные ВАЗ.

## Последствия

- Разбор мгновенный, офлайн, бесплатный, без ключей. Для Vesta/XRAY — марка +
  модель + двигатель + год; для прочих ВАЗ — марка + год; для иномарок — vPIC.
- РФ-специфика (моат) — в коде проекта, а не в стороннем сервисе.
- Таблицы ВАЗ покрывают только то, что есть в vininfo (Vesta, XRAY). Granta,
  Priora, Kalina и т.п. дают лишь марку — таблицы расширяются вручную по мере
  сверенных данных.
- Кузов/завод/КПП из VDS доступны в vininfo, но в доменную модель `Car` пока не
  заведены — сознательно, чтобы не оставлять полу-интегрированные поля. Это
  точка расширения, если появятся соответствующие поля в форме.
- vPIC-резерв требует сети и знает не все иномарки; год ориентировочный
  (30-летний цикл) — оба поля в форме редактируемы.
- Лицензия: таблицы — производная от vininfo (BSD-3), требуется сохранение
  копирайта/атрибуции. Отражено в комментарии кода и в
  [`docs/ecosystem.md`](../ecosystem.md).
