# Архитектурный журнал (ADR)

Журнал архитектурных решений проекта (Architecture Decision Records). Каждое
значимое решение фиксируется отдельной записью `NNNN-краткое-название.md`.

**Записи неизменяемы.** Решение не удаляют и не переписывают: при пересмотре
заводят новую запись, а прежней меняют статус на «Заменено ADR-NNNN». Так виден
не только текущий выбор, но и почему он менялся.

## Реестр решений

| №    | Решение | Статус |
|------|---------|--------|
| [0001](0001-key-architecture-decisions.md) | Ключевые архитектурные решения: свой app + Claude API, абстракции транспорта и провайдера, нативный Android | Принято |
| [0002](0002-http-llm-providers.md) | LLM-провайдеры (Claude, Cloud.ru) через HTTP вместо Anthropic SDK | Принято |
| [0003](0003-vin-decoding-offline-first.md) | Разбор VIN: офлайн-первый, детали ВАЗ из vininfo, vPIC-резерв | Принято |
| [0004](0004-manufacturer-pids-user-defined.md) | Заводские PID (мода 21/22): не сочиняем каталог, задаёт пользователь | Принято |

## Как добавить запись

1. Возьми следующий свободный номер (0002, 0003, …).
2. Скопируй [`template.md`](template.md) в `NNNN-краткое-название.md`.
3. Заполни контекст, решение и последствия.
4. Добавь строку в реестр выше.
5. Если новое решение отменяет прежнее — не правь старый файл: заведи новую
   запись, а у старой проставь статус «Заменено ADR-NNNN».

## Статусы

- **Предложено** — обсуждается, ещё не действует.
- **Принято** — действующее решение.
- **Заменено ADR-NNNN** — отменено более новой записью.
- **Устарело** — потеряло актуальность, прямой замены нет.
